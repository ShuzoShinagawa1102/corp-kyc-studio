# 銀行の法人KYC/オンボーディング / 01_overview

    ## ビジョン
    法人取引開始を、規制に耐えつつも顧客にとって摩擦の少ない体験へ変える。

    ## このドメインをどう見るか
    この領域の本質は、**書類やフォームを処理すること**ではない。  
    本質は、**意思決定に必要な正当性・証拠・ルール適合性・責任所在を、期限内に接続すること**である。

    したがって本テーマは、単なる自動化ツールではなく、次の問いに答える業務基盤として捉える。

    - いま何の案件が動いているか
    - その案件はどの規則・契約・証拠に拘束されるか
    - 次に誰が何をすべきか
    - 何が例外で、何が通常か
    - 顧客価値は「処理件数」ではなく、どの結果改善で測るべきか

    ## 対象ドメイン
    法人顧客オンボーディングは、商品別要求・国別規制・実質支配者確認・制裁スクリーニング・継続審査が分断され、審査メモ作成と例外処理が人手に集中している。

    ## 顧客と受益者
    - **一次顧客（予算保持者）**: 商業銀行、投資銀行、決済事業者、証券会社、RegTech/BPO事業者
    - **現場受益者**: 業務担当者、管理者、監査/統制担当
    - **最終受益者**: エンドユーザー、患者、取引先、契約者、発注者など、処理遅延や品質ばらつきの影響を受ける当事者

    ## 顧客が利益を被るメカニズム
    口座開設リードタイム短縮、必要資料欠落削減、リスクレビュー品質平準化、周期レビューからイベント駆動DDへの移行、監査対応強化

    重要なのは、「1人の作業を軽くする」だけで終わらないことだ。  
    このドメインでは、遅延や不整合が **売上、資金化、顧客体験、規制対応、監査対応** に波及する。  
    だからこそ、プロダクトの価値は UI の便利さではなく、**業務のボトルネックを外せるか** で決まる。

    ## なぜ10億円級が狙えるか
    McKinsey は新規法人顧客のオンボーディングが最大100日かかり得るとし、2025年には agentic AI の有力適用先として end-to-end KYC workflow を挙げている。PwC/TheCityUK は英国金融機関の規制遵守コストが平均営業費用の13%超に達しうると推計している。[R1][R2][R3]

    ## 主要アクター
    - **RM/営業**: 顧客獲得と初期情報取得を担う
- **KYCアナリスト**: 書類収集・レビュー・リスクメモ作成
- **コンプライアンス責任者**: 高リスク判断・承認責任
- **顧客企業担当者**: 会社情報・UBO・証憑提出
- **外部データ提供者**: 登記・制裁・PEP・ adverse media を供給

    ## 提供するべきプロダクトの姿
    この領域では、チャットUIや文書要約だけでは不十分である。必要なのは以下を備えた「業務OS」である。

    1. 案件・ケースの集約
    2. ルールと証拠の接続
    3. 例外の検知と優先順位付け
    4. 関係者へのアサインと追跡
    5. 監査証跡と再現性
    6. 学習ループ（過去案件を次回運用へ還元）

    ## 本資料におけるプロダクト方針
    - ドメイン用語を先に定義し、実装詳細は後ろに置く
    - 生成AIは一部機能であり、中心はケース運用と状態管理
    - 既存基幹システムを置き換えるより、**既存システム間の高摩擦部分を支配する**
